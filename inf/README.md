# Исследование проекта AlexGroup Catalog

## Введение

В рамках данного проекта была разработана fullstack-система для каталога объектов недвижимости. Основной задачей являлось создание удобного интерфейса для просмотра, фильтрации и сравнения объектов, а также реализация системы обработки пользовательских заявок.

Особое внимание уделялось архитектуре приложения, отказу от готовых CMS-решений и переходу к самостоятельной реализации backend-части с использованием современных инструментов.

---

## Цель проекта

Цель проекта — разработка современного веб-приложения, позволяющего:

- отображать каталог объектов недвижимости
- предоставлять пользователю инструменты фильтрации и сортировки
- реализовать сравнение объектов
- обеспечить возможность отправки заявок
- организовать хранение и обработку данных

---

## Этапы разработки

Разработка проекта проходила в несколько этапов:

### 1. Использование CMS

На начальном этапе рассматривалось использование CMS (Strapi) для управления данными. Были созданы модели объектов и выполнена базовая настройка API.

Однако в процессе работы выявились ограничения:
- недостаточная гибкость
- сложность интеграции
- избыточность функционала

В результате было принято решение отказаться от CMS.

---

### 2. Переход на Next.js

После отказа от CMS проект был полностью переведён на Next.js. Это позволило объединить frontend и backend в одном приложении.

Была реализована архитектура с использованием App Router и встроенных API маршрутов.

---

### 3. Реализация backend-логики

Вместо CMS была реализована собственная серверная часть:

- API для работы с объектами
- API для обработки заявок
- взаимодействие с базой данных через Prisma

---

### 4. Разработка интерфейса

На клиентской стороне были реализованы:

- каталог объектов
- карточки объектов
- страница объекта
- фильтры и сортировка
- сравнение объектов
- форма заявки

---

### 5. Работа с данными

Для хранения данных была выбрана PostgreSQL, а для взаимодействия — Prisma ORM.

Были созданы основные сущности:
- Object
- Lead

---

## Архитектура проекта

Проект реализован как fullstack-приложение.

### Клиентская часть

Отвечает за:
- отображение интерфейса
- взаимодействие с пользователем
- отправку запросов на сервер

### Серверная часть

Реализована внутри Next.js и выполняет:
- обработку API-запросов
- работу с базой данных
- бизнес-логику

---

## Основной функционал

### Каталог объектов
Позволяет пользователю просматривать список объектов с основными характеристиками.

### Фильтрация и сортировка
Обеспечивает удобный поиск по параметрам и настройку отображения.

### Страница объекта
Содержит подробную информацию и галерею изображений.

### Сравнение
Позволяет анализировать несколько объектов одновременно.

### Заявки
Реализует взаимодействие пользователя с системой через форму.

## Используемые технологии

### Next.js

Next.js — это React-фреймворк, который позволяет создавать как клиентскую, так и серверную часть приложения. В проекте использовался App Router, обеспечивающий современную маршрутизацию и поддержку серверных компонентов. Благодаря встроенным API routes удалось реализовать backend без отдельного сервера. Это значительно упростило архитектуру и ускорило разработку.

---

### React

React использовался для построения пользовательского интерфейса. С его помощью были реализованы компоненты каталога, карточек объектов и форм. Использование хуков позволило эффективно управлять состоянием и жизненным циклом компонентов. React обеспечил гибкость и переиспользуемость кода.

---

### TypeScript

TypeScript добавляет статическую типизацию в JavaScript. В проекте он использовался для повышения надёжности кода и предотвращения ошибок. Типизация особенно полезна при работе с API и базой данных. Это улучшает читаемость и поддержку проекта.

---

### Prisma

Prisma — это ORM для работы с базой данных. Она позволяет описывать структуру данных через схему и автоматически генерирует клиент для запросов. В проекте Prisma использовалась для выполнения CRUD-операций. Это упростило работу с PostgreSQL и сделало код более безопасным и понятным.

---

### PostgreSQL

PostgreSQL — это реляционная база данных, используемая для хранения информации об объектах и заявках. Она обеспечивает надёжность, масштабируемость и поддержку сложных запросов. В проекте используется для хранения основной бизнес-логики данных. Хорошо интегрируется с Prisma.

---

### Fetch API

Fetch API использовался для выполнения HTTP-запросов между клиентом и сервером. Он применяется для получения данных каталога, отправки заявок и обновления информации. Позволяет легко работать с асинхронными операциями. Является стандартным инструментом в современных веб-приложениях.
